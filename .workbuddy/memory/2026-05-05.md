### 问题2：数据库操作未等待初始化完成
**症状**：服务器启动失败，数据库操作报错"表不存在"

**原因**：`lib/store.js` 中的 `query()` 和 `run()` 函数没有等待数据库初始化完成就执行查询。

**修复**：修改 `query()` 和 `run()` 函数，在执行数据库操作前先等待 `dbInitPromise` resolve：
```javascript
function query(db, sql, params = []) {
  return waitForDbInit().then(() => {
    return new Promise((resolve, reject) => {
      db.all(sql, params, (err, rows) => {
        if (err) reject(err);
        else resolve(rows);
      });
    });
  });
}

function run(db, sql, params = []) {
  return waitForDbInit().then(() => {
    return new Promise((resolve, reject) => {
      db.run(sql, params, function(err) {
        if (err) reject(err);
        else resolve(this.lastID);
      });
    });
  });
}
```

**修改的函数**：

- `getUserById()` - 添加 `waitForDbInit().then(() => { ... })`

1. 后端 `server.js` 中 `GET /api/patients` 的 `id` 字段设置在 `...p` 展开之前，被 `p.id`（`undefined`）覆盖
2. `refreshPatientsByLevel()` 的编辑按钮事件缺少容错查找逻辑
3. `openPatientModal()` 中 `pmId.value` 在 `p.id` 为 `undefined` 时设为空串
4. 旧数据库 `patients` 表可能缺少 `idCard` 等字段（`CREATE TABLE IF NOT EXISTS` 不添加新列）

**修复内容**：

**修复1** `server.js` 第1088-1098行：`GET /api/patients` 的 `id` 放在 `...p` 之后
```javascript
const list = rawList.map((p) => {
  const safeId = p.id || p.patientNo || '';
  return { ...p, id: safeId, assessmentFrequencyDays: p.assessmentFrequencyDays ?? null };
});
```

**修复2** `public/app.js` `refreshPatientsByLevel()` 编辑按钮：添加容错查找逻辑（与 `refreshPatients()` 一致）

**修复3** `public/app.js` `openPatientModal()`：`pmId.value` 使用 `p.id || p.patientNo || ''` 兜底

**修复4** `lib/store.js`：新增 `migratePatientsTable()` 函数，自动为旧数据库添加缺失字段（`idCard`、`assessmentFrequencyDays` 等）

### 问题2：身份证号码丢失
**根本原因**：旧数据库 `patients` 表没有 `idCard` 字段，导致读取和保存时身份证号码丢失。

**修复**：`migratePatientsTable()` 会在服务器启动时自动检查并添加缺失的字段。

### 文件变更摘要
- `server.js` - 修改（`GET /api/patients` 确保 `id` 字段正确）
- `public/app.js` - 修改（`refreshPatientsByLevel()` 容错查找 + `openPatientModal()` 兜底）
- `lib/store.js` - 修改（新增 `migratePatientsTable()` 迁移函数）
