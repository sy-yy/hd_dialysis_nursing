# 血液透析患者分层分级管理系统部署指南

## 1. 环境准备

### 1.1 服务器要求
- 操作系统：Windows Server 或 Windows 10/11
- 内存：至少 2GB
- 存储空间：至少 10GB
- 网络：连接到内网，确保局域网内其他电脑能够访问

### 1.2 安装 Node.js
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载并安装 LTS 版本的 Node.js（推荐 v16 或更高版本）
3. 安装完成后，打开命令提示符，运行以下命令验证安装：
   ```
   node -v
   npm -v
   ```

## 2. 部署步骤

### 2.1 复制项目文件
1. 将项目文件夹 `hd_dialysis_nursing` 复制到服务器的合适位置，例如 `D:\Apps\hd_dialysis_nursing`

### 2.2 安装依赖
1. 打开命令提示符，进入项目目录：
   ```
   cd D:\Apps\hd_dialysis_nursing
   ```
2. 运行以下命令安装依赖：
   ```
   npm install
   ```

### 2.3 启动服务器
1. 推荐直接双击项目目录下的 `start.bat` 启动。

2. 也可以在项目目录下运行以下命令启动服务器：
   ```
   node server.js
   ```
   或者
   ```
   npm start
   ```

3. 如果在 PowerShell 中执行 `npm start` 遇到脚本执行策略拦截，优先使用以下方式：
   ```
   node server.js
   ```
   或直接运行 `start.bat`。

4. 服务器启动后，会在控制台显示：
   ```
   服务已启动，监听端口 3081
   数据库初始化完成
   ```

### 2.4 配置防火墙
1. 打开 Windows 防火墙设置
2. 点击「高级设置」
3. 点击「入站规则」→「新建规则」
4. 选择「端口」→「下一步」
5. 选择「TCP」，并在「特定本地端口」中输入 `3081` →「下一步」
6. 选择「允许连接」→「下一步」
7. 保持默认设置，点击「下一步」
8. 输入规则名称，例如「血液透析患者分层分级管理系统」→「完成」

## 3. 访问系统

### 3.1 获取服务器 IP 地址
1. 在服务器上打开命令提示符，运行：
   ```
   ipconfig
   ```
2. 找到「IPv4 地址」，例如 `192.168.1.100`

### 3.2 局域网访问
1. 在局域网内的任何电脑上，打开浏览器
2. 输入服务器 IP 地址和端口：
   ```
   http://192.168.1.100:3081
   ```
3. 即可访问血液透析患者分层分级管理系统

## 4. 系统维护

### 4.1 停止服务器
- 在命令提示符中按 `Ctrl+C` 停止服务器

### 4.2 重启服务器
- 重新运行 `start.bat`、`node server.js` 或 `npm start`

### 4.3 数据备份
- 定期备份项目目录下的 `.db` 文件，这些是系统的数据库文件

## 5. 常见问题

### 5.1 无法访问系统
- 检查服务器是否正在运行
- 检查防火墙是否已正确配置
- 检查局域网连接是否正常
- 尝试在服务器本地访问 `http://localhost:3081` 验证服务是否正常

### 5.2 数据库连接错误
- 检查项目目录是否有写入权限
- 检查 `.db` 文件是否被占用

### 5.3 启动失败
- 检查依赖是否正确安装
- 检查 Node.js 版本是否兼容
- 如果 PowerShell 提示禁止运行 `npm.ps1`，请改用 `node server.js` 或 `start.bat`
- 查看控制台错误信息并进行相应修复

## 6. 技术支持

如果遇到部署问题，请联系系统管理员或技术支持人员。
