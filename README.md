# 血液透析患者分层分级护理评估系统

## 项目简介

血液透析患者分层分级护理评估系统是一款专为血液透析中心设计的患者管理软件。系统采用现代化的 Web 技术栈，提供患者信息管理、护理评估评分、数据分析、语音识别录入等功能，帮助医护人员高效管理患者信息，实现分层分级个性化护理。

**版本**: 3.0.0
**更新日期**: 2026-05-19
**运行环境**: Windows 内网 B/S 架构

---

## 核心功能

### 1. 患者信息管理
- **完整的患者档案**：姓名、性别、出生日期、身份证号、联系电话、首次透析日期、身高、干体重等
- **身份证智能识别**：18位身份证号自动提取性别、出生日期，校验码严格验证，自动计算年龄
- **必填项验证**：姓名、身份证号码、联系电话、首次透析日期均为必填项，前后端双重验证
- **智能日期校验**：首次透析日期不能早于出生日期、不能晚于当前系统日期
- **年龄自动计算**：根据身份证号或出生日期自动计算并显示年龄

### 2. 护理评估评分
- **分层分级评估**：基于评分规则对患者的透析龄、合并症、营养状况、活动能力等进行综合评定
- **评估频率设置**：每个护理等级可配置不同的评估周期（天数），评估数据随级别设置自动同步更新
- **评分级别可视化**：低危/中危/高危等级用颜色区分，分数范围不允许重叠
- **自动提醒**：系统自动计算下次评估时间并在仪表盘提醒
- **评估记录追踪**：完整记录历史评估数据，支持按患者、等级、日期筛选

### 3. 数据导入导出
- **Excel 批量导入**：支持通过 Excel 模板批量导入患者数据，自动识别列名
- **导入结果弹窗**：导入完成后弹窗展示成功/失败数量及每条失败记录的行号与原因
- **导入数据校验**：身份证校验码验证、首次透析日期必填及合理性校验、重复身份证检测
- **日期格式兼容**：支持 `2023-01-01`、`2023/5/4`、`2023.5.4` 等多种日期格式，Excel 原生日期自动转换
- **模板一键下载**：导出含必填标记（★）和示例数据（※）的 Excel 模板
- **数据导出**：支持导出患者列表和评估记录

### 4. 语音识别服务
- **讯飞开放平台集成**：对接讯飞语音识别 API，支持语音录入
- **透析护理场景适配**：针对透析护理场景优化语音录入体验

### 5. 数据查询与统计
- **多条件搜索**：支持按姓名、编号、床号等快速查找
- **分级筛选**：按护理等级筛选患者列表
- **统计仪表盘**：患者分布、评估完成情况、分级占比等统计图表
- **操作日志**：记录所有关键操作，便于审计追溯

### 6. 系统管理
- **多用户支持**：支持医师、护士等多角色账号，注册审批机制
- **权限管理**：不同角色拥有不同操作权限（查看、管理、评分、导入、系统设置等）
- **数据备份**：支持一键备份整个程序目录（WinRAR）
- **系统设置**：可配置评估评分规则、级别颜色、字段显示等参数

## 快速部署

### 环境要求
- **操作系统**: Windows Server 或 Windows 10/11
- **内存**: 至少 2GB
- **存储空间**: 至少 500MB（不含数据）
- **网络**: 连接到内网，确保局域网内其他电脑能够访问
- **Node.js**: v16 或更高版本

### 安装步骤

#### 1. 安装 Node.js
1. 访问 [Node.js 官网](https://nodejs.org/) 下载并安装 LTS 版本
2. 验证安装：
   ```bash
   node -v
   npm -v
   ```

#### 2. 部署项目
1. 将项目文件夹复制到服务器，例如 `D:\Apps\hd_dialysis_nursing`
2. 打开命令提示符，进入项目目录：
   ```bash
   cd D:\Apps\hd_dialysis_nursing
   ```
3. 安装依赖：
   ```bash
   npm install
   ```

#### 3. 启动服务
**推荐方式**：双击 `start.bat`

**命令行方式**：
```bash
node server.js
```

成功启动后，控制台会显示：
```
服务已启动，监听端口 3081
数据库初始化完成
```

#### 4. 配置防火墙
1. 打开 Windows 防火墙 → 高级设置
2. 入站规则 → 新建规则
3. 选择「端口」→ TCP → 特定本地端口：`3081`
4. 允许连接 → 完成

## 系统维护

### 日常维护
1. **定期检查日志**：查看 `logs/` 目录下的日志文件

### 常见问题

- 检查服务器是否正在运行
- 检查防火墙是否已正确配置
- 检查局域网连接是否正常

- 检查项目目录是否有写入权限
- 检查 `.db` 文件是否被占用
- 尝试重启服务器

- 检查依赖是否正确安装（运行 `npm install`）
- 检查 Node.js 版本是否兼容（v16+）
- 查看控制台错误信息并进行相应修复
- PowerShell 脚本执行问题，请使用 `node server.js` 或 `start.bat`

## 技术架构

### 前端
- **框架**: 原生 JavaScript (Vanilla JS)
- **UI**: 自定义 CSS，响应式设计
- **特性**: SPA 单页应用，无刷新操作
- **Excel 处理**: SheetJS (xlsx.js) 客户端生成导入模板

### 后端
- **运行环境**: Node.js
- **Web 框架**: Express.js
- **数据库**: SQLite3
- **认证**: Session-based 会话管理 (express-session + bcryptjs)
- **文件上传**: multer
- **Excel 解析**: SheetJS (xlsx) 服务端解析导入
- **语音识别**: 讯飞开放平台

| 文件 | 说明 |

| `xlsx.full.min.js` | SheetJS（xlsx.js）客户端压缩版，用于前端生成 Excel 导入模板和导出数据文件 |

##### `public/patch-*.js` — 运行时补丁脚本

| 文件 | 说明 |
|------|------|
| `patch-refresh-dashboard.js` | 仪表盘刷新补丁，改用动态计算统计数据（不依赖存储字段），确保统计数字实时准确 |
| `patch-refresh-patients-by-level.js` | 按级别筛选患者列表补丁，使用 `/api/patients/with-assessments` 端点动态计算评估信息，修复从仪表盘点击分级标签后患者列表级别显示不正确的问题 |
| `patch-refresh-patients.js` | 患者列表刷新补丁，使用 `/api/patients/with-assessments` 端点动态计算评估信息，确保列表中的评估状态实时更新 |
| `patch-refresh-settings.js` | 设置页面刷新补丁，集成 `level-manager.js` 中的可视化渲染函数，替代原有的简单文本渲染 |

> **补丁机制说明**：这些 patch 脚本在页面加载后覆盖 `app.js` 中的原始函数，用于快速修复问题而无需修改庞大的 `app.js` 主文件。在生产稳定后，补丁逻辑应合并回 `app.js`。

#### `data/` — 数据存储目录

| 文件 | 说明 |
|------|------|
| `patients.db` | 患者数据库（SQLite），存储患者基本信息（姓名、身份证号、联系电话、首次透析日期、身高、干体重等）及动态字段配置 |
| `assessments.db` | 评估数据库（SQLite），存储护理评估记录，包含评分详情、评估日期、评估人、护理等级等信息 |
| `users.db` | 用户数据库（SQLite），存储系统用户账号信息，包括用户名、密码哈希（bcrypt）、角色权限、注册审批状态等 |
| `rubric.json` | 评分规则配置文件（JSON），定义评分项、各级别的分数范围、颜色、评估周期、描述等参数，由评分级别设置界面管理 |

> **备份提示**：定期备份此目录下的 `.db` 文件即可保全所有业务数据。`rubric.json` 可通过系统界面重新生成，但建议一并备份。

#### `logs/` — 日志文件目录

| 文件 | 说明 |

> 日志文件按日自动轮转，无需手动清理。如需排查问题，查看对应日期的日志文件即可。
