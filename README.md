# 血液透析患者分层分级管理系统

## 📋 项目简介

血液透析患者分层分级管理系统是一款专为血液透析中心设计的患者管理软件。系统采用现代化的Web技术栈，提供患者信息管理、护理评估、数据分析等功能，帮助医护人员高效管理患者信息，实现个性化护理。

**版本**: 2.0.0  
**更新日期**: 2026-05-11  
**技术支持**: 血液透析护理团队

---

## ✨ 核心功能

### 1. 患者信息管理
- ✅ **完整的患者档案**：姓名、性别、出生日期、身份证号、联系电话等
- ✅ **必填项验证**：患者姓名、身份证号码、联系电话、首次透析日期均为必填项
- ✅ **智能日期校验**：首次透析日期不能早于出生日期，不能晚于当前系统日期
- ✅ **身份证智能识别**：自动从身份证号码提取性别、出生日期、年龄
- ✅ **动态字段支持**：支持自定义患者信息字段（后端已部署，前端按需启用）

### 2. 护理评估管理
- 📊 **分层分级评估**：根据患者情况进行护理等级评定
- 📅 **评估频率设置**：可自定义每个患者的评估频率（天数）
- ⏰ **自动提醒**：系统自动计算下次评估时间并提醒
- 📈 **评估记录追踪**：完整记录历史评估数据

### 3. 数据查询与统计
- 🔍 **多条件搜索**：支持按姓名、编号、床号等快速查找
- 📊 **数据筛选**：按性别、护理等级、透析频率等维度筛选
- 📈 **统计报表**：患者分布情况、评估完成情况统计
- 📤 **数据导出**：支持导出患者列表和评估记录

### 4. 系统管理
- 👥 **多用户支持**：支持医师、护士等多角色账号
- 🔐 **权限管理**：不同角色拥有不同的操作权限
- 📝 **操作日志**：记录所有关键操作的日志，便于审计
- ⚙️ **系统设置**：可配置评估标准、字段显示等参数

## 🚀 快速部署

### 环境要求
- **操作系统**: Windows Server 或 Windows 10/11
- **内存**: 至少 2GB
- **存储空间**: 至少 10GB
- **网络**: 连接到内网，确保局域网内其他电脑能够访问
- **Node.js**: v16 或更高版本

### 安装步骤

#### 1. 安装 Node.js
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载并安装 LTS 版本的 Node.js
3. 验证安装：
   ```bash
   node -v
   npm -v
   ```

#### 2. 部署项目
1. 将项目文件夹 `hd_dialysis_nursing` 复制到服务器，例如 `D:\Apps\hd_dialysis_nursing`
2. 打开命令提示符，进入项目目录：
   ```bash
   cd D:\Apps\hd_dialysis_nursing
   ```
3. 安装依赖：
   ```bash
   npm install
   ```

#### 3. 启动服务
**推荐方式**：直接双击运行 `start.bat`

**命令行方式**：
```bash
# 方式1：使用 npm
npm start

# 方式2：直接运行 Node
node server.js
```

**PowerShell 脚本执行策略问题**：
如果在 PowerShell 中遇到脚本执行被阻止，请使用：
```bash
node server.js
```
或直接运行 `start.bat`

成功启动后，控制台会显示：
```
服务已启动，监听端口 3081
数据库初始化完成
```

#### 4. 配置防火墙
1. 打开 Windows 防火墙 → 高级设置
2. 入站规则 → 新建规则
3. 选择「端口」→ TCP → 特定本地端口：`3081`
4. 允许连接 → 完成

## 🌐 访问系统

### 获取服务器 IP 地址
在服务器上运行：
```bash
ipconfig
```
找到 IPv4 地址，例如 `192.168.1.100`

### 访问方式
1. **本地访问**：`http://localhost:3081`
2. **局域网访问**：`http://192.168.1.100:3081`（替换为实际IP）

## 📂 数据备份

### 自动备份
系统支持一键备份功能，可在系统设置中进行备份操作。

### 手动备份
#### 备份数据库文件
定期备份项目目录下的 `.db` 文件（SQLite 数据库）

#### 备份整个程序目录
使用 WinRAR 备份整个项目目录：
```powershell

```

## 🔧 系统维护

### 日常维护
1. **定期检查日志**：查看 `logs/` 目录下的日志文件
2. **定期备份数据**：建议每周备份一次数据库
3. **监控磁盘空间**：确保系统有足够存储空间

### 常见问题

#### 无法访问系统
- ✅ 检查服务器是否正在运行
- ✅ 检查防火墙是否已正确配置
- ✅ 检查局域网连接是否正常
- ✅ 尝试本地访问 `http://localhost:3081`

#### 数据库连接错误
- ✅ 检查项目目录是否有写入权限
- ✅ 检查 `.db` 文件是否被占用
- ✅ 尝试重启服务器

#### 启动失败
- ✅ 检查依赖是否正确安装（运行 `npm install`）
- ✅ 检查 Node.js 版本是否兼容（v16+）
- ✅ 查看控制台错误信息并进行相应修复
- ✅ PowerShell 脚本执行问题，请使用 `node server.js` 或 `start.bat`

## 📊 技术架构

### 前端技术栈
- **框架**: 原生 JavaScript (Vanilla JS)
- **UI**: 自定义 CSS，响应式设计
- **特性**: SPA 单页应用，无刷新操作

### 后端技术栈
- **运行环境**: Node.js
- **Web 框架**: Express.js
- **数据库**: SQLite (better-sqlite3)
- **认证**: Session-based 会话管理

### 项目结构
```
hd_dialysis_nursing/
├── server.js              # 后端服务器主文件
├── public/               # 前端静态资源
│   ├── index.html        # 主页面
│   ├── app.js            # 前端交互逻辑
│   └── app.css          # 样式文件
├── data/                 # 数据库文件
│   └── *.db
├── logs/                 # 日志文件
├── package.json          # 项目配置
├── start.bat             # 启动脚本
└── README.md             # 项目文档
```
